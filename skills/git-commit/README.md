# Git Commit

一个 Claude Code 技能，帮你生成规范化的 Git 提交信息。它会读取你的代码改动、理解改动意图，生成符合 [Conventional Commits](https://www.conventionalcommits.org/) 规范的提交信息，你确认后才执行提交。

## 特性

- **意图优先** — 不只是看 diff 文本，而是理解改动背后的目的。删除一个函数可能是"移除废弃的 API"，而不是"删了点代码"
- **规范提交** — 自动生成 `feat`、`fix`、`refactor` 等标准 type + 中文描述的提交信息
- **确认再提交** — 展示建议的提交信息，你来决定是确认、修改还是取消
- **状态感知** — 自动区分暂存区和工作区的改动状态，给出合理的处理方式

## 安装

```bash
npx skills add zxc7563598/skillbox@git-commit
```

安装后，在 Claude Code 会话中直接说「提交代码」或「帮我生成 commit message」即可触发。

## 使用方式

```bash
# 触发方式一：直接执行
/git-commit

# 触发方式二：带提示
/git-commit 修了登录页验证码刷不出来的问题
```

技能会执行以下流程：

1. **读取改动** — 获取 `git diff`，了解暂存区和工作区的状态
2. **理解意图** — 分析改动之间的关联，推断这次改动的真实目的
3. **生成信息** — 输出 Conventional Commits 格式的建议提交信息
4. **等你确认** — 你点击「确认提交」、「我要修改」或「取消」

## 提交信息格式

```
<type>(<scope>): <中文主题>

<中文正文>
```

示例：

```
feat(auth): 添加 JWT 令牌认证功能

新增 JWT 验证中间件，在路由层拦截未认证请求。
支持令牌过期自动刷新。
```

```
fix(user): 修复分页总数计算错误

分页总数之前包含了已删除的用户，导致列表页空页过多。
现在使用 COUNT(*) 并过滤 deleted_at。
```

## 常见问题

### 和直接 `git commit -m "xxx"` 有什么区别？

自己手写容易写成「修了个 bug」「改了点东西」，回头看提交历史时不知道当时干了什么。这个技能会帮你理清改动的意图，生成结构化的提交信息。

### 生成的提交信息我不满意怎么办？

确认步骤会提供「我要修改」选项，你可以输入修改意见，技能会基于反馈重新生成。

### 改动很大怎么办？

当 diff 超过 500 行时，技能会按文件分组分析，如果判断应该拆成多个提交，会主动建议你拆分。
